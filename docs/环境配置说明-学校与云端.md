## 一、环境前先确认这些

### 1. 权限与隔离（必看）

- **不要用系统自带的 Python / pip**：你当前环境 `pip` 无权限写系统目录，必须用**独立环境**。
- **推荐 Conda**：用 conda 创建独立环境，所有包装在用户目录，不依赖管理员。
- 若学校只提供 **module**（如 `module load cuda/12.8`），先 `module load` 再创建 conda 环境，保证 `nvcc`、驱动一致。

### 2. GPU 与驱动

| 项目 | 要求 |
|------|------|
| **CUDA** | ≥ 12.8（官方推荐） |
| **驱动** | 与 CUDA 匹配（如 570+ for CUDA 12.8） |
| **显卡** | NVIDIA（推荐）；AMD 用 ROCm 需单独看 [verl AMD 文档](https://github.com/verl-project/verl/blob/main/docs/start/install.rst) |

在登录节点执行查看：

```bash
nvidia-smi          # 看驱动、CUDA 版本、显卡型号
nvcc --version      # 若有 toolkit，看编译器版本
```

若集群是**提交作业到计算节点**，要在作业脚本里同样 `module load` 或保证计算节点有相同 CUDA 环境。

**登录节点无 GPU 时：进 Slurm GPU 节点（交互式）**

在登录节点装依赖（如 flash-attn）若报 `libcuda.so.1: cannot open shared object file`，说明当前节点没有 GPU 驱动。可先申请一个带 GPU 的**交互式作业**，在计算节点上装：

```bash
# 申请 1 卡、1 小时，得到计算节点上的 shell（分区/账户与 run_benchmark.sh 一致）
srun --partition=gpu --account=p201251 --qos=default --gres=gpu:1 --time=01:00:00 --pty bash -l
```

进入后执行 `nvidia-smi` 确认有显卡，再激活环境并安装。

**若集群计算节点无 nvcc/CUDA_HOME**，可用预编译 wheel 在登录节点直接装（无需 GPU）：

```bash
cd /project/home/p201251/hyl_ppo
source activate_verl.sh
# flash-attn：与 torch 2.6 + cu124 + Python 3.12 匹配的预编译包（无需编译）
pip install "https://github.com/mjun0812/flash-attention-prebuild-wheels/releases/download/v0.7.16/flash_attn-2.6.3+cu124torch2.6-cp312-cp312-linux_x86_64.whl"
# FlashInfer（vLLM 等可用）
pip install flashinfer-python
```

装完后，之后 `sbatch` 提交的任务会自动用到。**启用 FlashInfer 采样**：在跑 vLLM 的脚本里设置 `export VLLM_USE_FLASHINFER_SAMPLER=1`（如 `run_benchmark.sh` 已包含），可加速 top-k/top-p 采样。

### 3. 存储与配额

- **模型**：Qwen3-4B-Base 约 **7.5GB**，verl 依赖+缓存还会占不少，建议工作目录至少 **30GB+** 可用。
- **检查**：`df -h $HOME` 或 `df -h /project/home/p201251`。
- 若配额紧张：把大模型放到共享存储或 scratch，用软链到当前项目目录。

### 4. 网络（学校/国内常见）

- **Hugging Face**：若 `huggingface-cli download` 或 `git clone https://huggingface.co/...` 很慢或超时，可设镜像（需学校/云支持）：
  ```bash
  export HF_ENDPOINT=https://hf-mirror.com
  ```
- **pip 源**：建议用国内源加速：
  ```bash
  pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
  ```
- 若**不能访问外网**：需在能联网的机器用 `pip download` 下好 wheel，再拷到集群离线安装。

## 二、推荐配置步骤（Conda + FSDP，单机多卡）

以下按「先推理栈、再训练栈、最后 verl」顺序，减少版本被覆盖。

### 1. 创建 Conda 环境

```bash
# Python >= 3.10，官方推荐 3.12
conda create -n verl python=3.12 -y
conda activate verl
```

### 2. 加载集群 CUDA（若有 module）

```bash
module load cuda/12.8    # 名称以你们集群为准
# 或 module load gcc/xx 如编译需要
```

### 3. 先装推理引擎与 PyTorch（避免被后续覆盖）

verl 支持 **vLLM** 或 **SGLang** 做 rollout。先装其一，再装其它依赖。

**方案 A：仅 vLLM（常用）**

```bash
pip install torch --index-url https://download.pytorch.org/whl/cu128
pip install vllm>=0.8.2   # 文档建议 0.8.3+，脚本里为 0.11.0
# 可选：export VLLM_USE_V1=1 提升性能
```

**方案 B：用 verl 官方脚本（vLLM + 可选 SGLang/Megatron）**

仅 FSDP 训练、不用 Megatron 时：

```bash
cd /project/home/p201251/hyl_ppo/verl
USE_MEGATRON=0 bash scripts/install_vllm_sglang_mcore.sh
```

脚本会装 SGLang、vLLM、FlashAttention、PyTorch 等；若你**只用 vLLM**，可先不装 SGLang，再视需要改脚本或手动装 vLLM。

### 4. 安装 verl 本体（可编辑安装）

```bash
cd /project/home/p201251/hyl_ppo/verl
pip install --no-deps -e .
```

`--no-deps` 避免覆盖上一步装好的 torch/vLLM。

### 5. 补全依赖（若脚本未装全）

```bash
pip install accelerate datasets peft "transformers>=4.51.0" \
    "numpy<2.0.0" "pyarrow>=15.0.0" pandas "tensordict>=0.8.0,<=0.10.0,!=0.9.0" \
    "ray[default]" hydra-core wandb packaging
```

**Qwen3 必须** `transformers>=4.51.0`，否则会 `KeyError: 'qwen3'`。

### 6. Flash Attention（按需）

- 有 GPU 且 CUDA 正常时建议装，加速且省显存：
  ```bash
  pip install flash-attn --no-build-isolation
  ```
- 若编译失败或环境受限，可先不装，部分功能会回退到非 flash 实现。

### 7. 验证环境

```bash
python -c "
import torch; print('CUDA:', torch.cuda.is_available(), torch.version.cuda)
import transformers; print('transformers:', transformers.__version__)
from verl import __version__; print('verl:', __version__)
# 若装了 vLLM：
import vllm; print('vllm:', vllm.__version__)
"
```

## 三、学校/云端特别注意事项

1. **作业调度**  
   若用 Slurm/PBS 等，训练命令要在**作业脚本**里跑，并申请足够 GPU、内存、时间；`ray` 的节点数要跟实际申请的一致。

2. **多机多卡**  
   多节点时需正确设置 `MASTER_ADDR`、`MASTER_PORT`、`NODE_RANK` 等；verl 文档与示例里有 Ray 的 multi-node 说明。

3. **显存与 batch**  
   Qwen3-4B 单卡可跑，但 PPO/GRPO 通常 actor + critic + rollout，建议至少 1 张 24GB 以上显卡；更大 batch 或更大模型需多卡或 FSDP。

4. **长任务断连**  
   用 `tmux` 或 `screen` 跑长训练，避免 SSH 断开会话导致进程被杀。

5. **数据与 checkpoint**  
   数据放共享存储或项目目录；checkpoint 路径统一写到当前项目或大容量目录，避免写满 home。
